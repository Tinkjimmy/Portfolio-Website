Case Study: myFlix

By Giacomo Duca

About the Project

Movie app is a single-page, responsive web application that allows users to registrate,update personal data, browse through a rich movies list and saving movies in a user specific favorite movie list.

Purpose & Context

This application was meant to be part of my web development course, to both exercise and show my newly acquired skills In full-stack javascript development.

Objective

The objective was to create a full project to put on display on my portfolio-website. The challenge was to build the application from the ground up both in the back and in the front-end part of it. The use of React was meant to give the application scalability an a better UI.

Technologies

Utilizes Postman, React, Bootstrap, Heroku, and the MERN Stack.

Server Side

On the server side I used Node.js and Express to create a REST API. The API was used to communicate with a MongoDB database(Mongoose was used to interact with the database) using GET and POST methods to fetch and save data from and to it.

Setup

The choice of a non relational database granted me the flexible setting I needed, allowing me to store data in a more dynamic format.

Models

I then drew models (despite MongoDb being a schema-less database) in order to facilitate efficient data storage, retrieval, and updating. I created API end points and defined routes and handlers for each of them.

Authentication & authorization

Authentication and authorization was an important step to secure the application, I installed packages for basic HTTP authentications(for initial login) and JWT(JSON Web Token) authentication using Passport(to authenticate and authorize subsequent requests after the initial login). An ulterior layer of security was put in place by adding Password hashing in my user schema ans integrating CORS to check which domains were allowed to make requests.

Deployment

I then deployed the API on Heroku , while the database was uploaded to MongoDB Atlas and i performed a final round of testing using Postman .

Client Side

I designed the client side as a single-page application developed using React and React-redux. The UI allows the user to view a list of movies and info about them, register through the sign up view, login and review his personal information in the profile view.

Steps

Approach

I designed the client side as a single-page application developed using React and React-redux. The UI allows the user to view a list of movies and info about them, register through the sign up view, login and review his personal information in the profile view.

Style

I utilized React Bootstrap to style the app, making use of its components such as Cards and Forms, and leveraging the grid system for layout and responsiveness.

MVC Architecture

I chose React as a framework to create the client side so that I could get familiar with the MVC architecture as a design pattern, also tacking advantage of its flexibility, rich ecosystem and strong community support. Learning how to build components in this environment was extremely useful since components allow for a modular approach to building the UI, making the codebase easier to manage and extend.

Final product

The last step was to implement Redux, to learn how it manages state changes, making them predictable through a unidirectional data flow. I then hosted the project on Netlify.

Retrospective

What went well

the initial phase of creating the API went exceptionally well. The clear definition of endpoints, models, and the thoughtful structuring of the database laid a strong foundation for the project. Leveraging Node.js and Express made the backend robust and scalable.

Challenges

Time Management

Juggling full-time work while dedicating sufficient time to the project was the most challenging aspect. Finding balance often meant sacrificing weekends and evenings to maintain project momentum.

Learning Curve

Grasping new concepts, especially Redux and advanced React patterns, required a considerable investment of time and effort.

Future improvement

More emphasis could be placed on implementing automated tests from the early stages. Testing could enhance reliability and facilitate easier refactoring and feature additions. The UI could also benefit from a restyling

Fianl Thoughts

The experience of developing the myFlix app, from backend to frontend, has been immensely rewarding. It has not only solidified my understanding of full-stack development principles but also enhanced my problem-solving skills and ability to learn and adapt to new technologies rapidly. Balancing this project with full-time work responsibilities underscored the importance of time management, perseverance, and the value of practical, hands-on learning.
